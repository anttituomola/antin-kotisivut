# mail-list

Mailing-list service for anttituomola.fi. Runs on the UpCloud server
(80.69.173.166) as a pm2 process behind nginx, with SQLite for storage.

## How it works

1. Visitors subscribe via the `SubscribeBox` component on the site. The form
   POSTs to `/mail-api/subscribe`, an Astro proxy route
   (`src/pages/mail-api/[...path].ts`) that forwards to this service — so the
   browser never talks to the backend directly.
2. Subscribers are stored as `pending` and get a confirmation email
   (double opt-in). `GET /api/confirm?token=...` activates them.
3. The service polls the site's blog feed (`FEED_URL`,
   `/blog/feed.json` generated by `src/pages/blog/feed.json.ts`) every
   `POLL_INTERVAL_MINUTES` minutes. A post that appears in the feed for the
   first time is emailed to confirmed subscribers `SEND_DELAY_MINUTES` after
   detection — one individual AWS SES email per subscriber, in the
   subscriber's chosen language (fi/en, falling back to whatever exists).
   On the very first poll, all existing posts are marked as already sent.
4. Every email has a one-click unsubscribe link
   (`/api/unsubscribe?token=...`, RFC 8058).

## Endpoints

- `POST /api/subscribe` — `{ email, language: "en"|"fi", website (honeypot), source }`
- `GET /api/confirm?token=...`
- `GET|POST /api/unsubscribe?token=...` (GET shows a button so mail scanners
  don't unsubscribe people by prefetching)
- `GET /api/health`

## Configuration (`.env` on the server)

| Variable | Default | Notes |
| --- | --- | --- |
| `PORT` | `3004` | |
| `PUBLIC_BASE_URL` | — | Public URL of this service, used in email links |
| `FEED_URL` | site feed | |
| `FROM_EMAIL` / `FROM_NAME` | — | Must be a verified SES identity |
| `SEND_DELAY_MINUTES` | `30` | Delay between detecting a post and emailing it |
| `POLL_INTERVAL_MINUTES` | `5` | |
| `AWS_REGION`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY` | — | Without these, emails are only logged |
| `DRY_RUN` | `false` | `true` = log emails instead of sending |

## Operations on the server

```bash
ssh root@80.69.173.166
pm2 status mail-list
pm2 logs mail-list
sqlite3 /var/www/mail-list/data/mail.db "SELECT email, language, status FROM subscribers;"
```

Deploying changes: copy `server.js` to `/var/www/mail-list/` and
`pm2 restart mail-list`. nginx vhost: `/etc/nginx/sites-available/mail-list`
(certbot-managed TLS). SES must be out of sandbox to email arbitrary
subscribers.
